The two .bb files provided in the Yocto folder are reference examples derived directly from this project. Their purpose is to demonstrate how to perform cross-compilation on an x86 host to build target executables that can run on ARM-based platforms. These files can be regarded as core examples within the project.

In general, all you need to do is package the data structures generated by the tool (for example, all files under output/gen/cpp_out), integrate them into your own program, and write a corresponding .bb file. With this setup, you can cross-compile and run the program on your target platform.

The .bb files provided here serve only as examples. They illustrate how to cross-compile the C or C++ test programs located in the test directory to run on the target board. During this process, bitbake is used to invoke protoc to generate the schema data structures, which are then compiled together with the test scripts. It is important to emphasize that this is not how you should build your schema in practice. The recommended approach is to directly use the data structures generated by the tool, package them, integrate them into your own application, and then write the appropriate .bb file for cross-compilation on your target platform.

Note: If you apply these example .bb files directly to your environment, there is a high chance that the compilation will fail. You will need to adjust the configurations according to the development manual of your MPU platform.
